# 利用上週加入的全班的倉儲,在裡面編寫前10週的來統整心得
因為每個人都可以在上面做更新更改, 這樣就會出現操作上的問題，push得時候只能一個人push,太多人同時推會導致網站亂掉,為了避免此現象的發生,在個人或各組完成階段性改版之前,先透過獨立分支進行資料改版提交與推送,等個人階段性版本確定後, 再與分組分支合併, 最後各組的階段性版本再與建立網頁的主分支進行合併。
